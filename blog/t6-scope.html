What is 'scope' and how does it work in JavaScript?

Ok, here's my interpretation. But first, a declaration (pun intended). I have googled the term, watched videos, read articles, scanned Stack Overflow but this blog post is purely my own definition. So apologies for misleading you.

There are two types of scope in JavaScript
Global Scope

When you create but do not declare a variable, your code will be read and 'executed' at top level, throughout the whole programme.

JavaScript 'thingy' will read and execute your code as a blanket rule, applying the same variable throughout your programme. YOur variable will be same value every where on your programme (the Window level).

Local Scope

When you create and declare (assign value) a variable within a function, or a method, then the JavaScript 'thingy' will look for and execute it at that level (the Document level).

DOcument belongs to the Window

In other words:

  • Declare your variables as you create them
  • If not feasible, then create them at function level
  • Local scope will be applicable to the function that is created, in a container of its own (nothing else can read that variable other than the function it is created in)
  • Global scope over-rides local scope
